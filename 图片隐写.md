# CTF 图片隐写术总结笔记

### 1. 基础属性与结构分析 (文件外部)
* **元数据 (Metadata)：** 检查 Exif 信息（备注、作者、GPS、缩略图）。
    * *工具：* `exiftool`
* **字符串搜索：** 检索文件中的可读文本（搜索 `flag`, `ctf`, `key`）。
    * *工具：* `strings`
* **文件附加：** 在图片结束标识符（如 JPG 的 `FF D9`）后拼接数据。
    * *工具：* `binwalk` (扫描), `foremost` / `dd` (提取)。
* **文件头与宽高：** 修复损坏的文件头，或修改 IHDR/SOF0 字节以显示被隐藏的底部区域。
    * *工具：* `010 Editor`



### 2. 空间域隐写 (无损格式 PNG/BMP)
* **LSB (最低有效位)：** 修改像素 RGB 值的最后一位，肉眼不可见。
    * *工具：* `StegSolve` (手动切通道), `zsteg` (自动化扫描)。
* **调色板 (Palette)：** 利用 `PLTE` 数据块的颜色索引或顺序隐藏信息。
* **通道隐藏：** 信息仅隐藏在特定通道（如 Alpha 透明通道）。



### 3. 变换域隐写 (有损格式 JPG)
针对像素的 DCT 系数或压缩算法进行的隐写，`binwalk` 无法检测。
* **Steghide：** 基于图论算法嵌入，通常需要解密密码。
    * *命令：* `steghide extract -sf img.jpg -p password`
* **OutGuess：** 针对 DCT 系数的隐写工具，常见于需要 Key 的题目。
    * *命令：* `outguess -k "key" -r img.jpg out.txt`
* **JPHide / JPHS：** 经典的 Windows 平台 JPG 隐写工具。



### 4. 格式特定隐写
* **GIF 隐写：**
    * 利用**帧间隔时间**（Delay Time）进行二进制编码。
    * 利用**逐帧拆分**，将 Flag 分散在不同帧中。
* **PNG IDAT 隐写：** 检查 IDAT 数据块是否异常、顺序是否颠倒或存在非图像数据的 IDAT 块。

### 5. 综合解密思路
* **Key 的来源：** 题目描述、图片 Exif、隐藏的压缩包内文件（如 `key.txt`）、脑洞（如 JOJO 替身名）。
* **常用流程：**
    1.  `exiftool` 查属性。
    2.  `strings` 搜字符。
    3.  `binwalk` 扫结构（若有隐藏包，提取后尝试用 Key 解压）。
    4.  `zsteg` / `StegSolve` 扫 LSB。
    5.  `steghide` / `outguess` 尝试 Key 提取算法隐写。
    6.  `010 Editor` 检查宽高。
