
```url
http://ca.gxu.edu.cn:81/zfim/system/getYhjsqx.zf
```

该HTTP请求/响应记录展现了一个**统一身份认证系统的用户权限与个人信息查询接口**。

| 项目          | 内容                                                   |
| :---------- | :--------------------------------------------------- |
| **接口路径**    | `/zfim/system/getYhjsqx.zf`                          |
| **推测含义**    | `getYhjsqx` → **获取用户角色权限**（拼音首字母缩写）                  |
| **核心作用**    | 在用户登录统一身份认证平台后，展示当前用户的**个人基本信息、所属角色、可访问的应用列表、权限功能**。 |
| **HTTP 方法** | `GET`                                                |
| **依赖状态**    | 依赖 Cookie 维持登录会话（`JSESSIONID` + 多种自定义 Cookie）。       |

该接口是认证中心（IDP）的**自服务面板**，用户登录后访问此页面可以查看自己在平台中的“数字身份”全貌。

| 头部字段                        | 值                                                   | 作用/风险提示                                                                                                                |
| :-------------------------- | :-------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------- |
| `Host`                      | `ca.gxu.edu.cn:81`                                  | 广西大学认证中心服务器，端口 81 为非标准 Web 端口。                                                                                         |
| `Cookie`                    | 包含 `JSESSIONID`、`iPlanetDirectoryPro` 等多项           | **会话凭证核心**。`JSESSIONID` 为 Tomcat 标准会话标识；`iPlanetDirectoryPro` 为 Oracle 身份管理产品线常见令牌，表明可能使用 Oracle Access Manager 或类似组件。 |
| `Referer`                   | `http://ca.gxu.edu.cn:81/zfim/system/login/main.zf` | 来源为登录主页。可用于防盗链或简单的 CSRF 防御参考。                                                                                          |
| `Upgrade-Insecure-Requests` | `1`                                                 | 浏览器期望升级至 HTTPS，但**本次通信仍为明文 HTTP**（非加密）。                                                                                |

Cookie 中包含多项复杂编码的字符串，例如 `_pv0`、`_pf0`、`ysydOtp` 等。这表明系统可能使用了：
- **Oracle Access Manager (OAM)** 或 **Oracle Identity Manager** 的加密 Cookie。
- 自定义的**多因子会话保持机制**。

| 头部字段                          | 值                                                           | 说明                                                                                   |
| :---------------------------- | :---------------------------------------------------------- | :----------------------------------------------------------------------------------- |
| `Server`                      | `none`                                                      | 隐藏了真实服务器软件，是一种安全加固措施（虽然下文 `X-Powered-By` 暴露了）。                                       |
| `X-Powered-By`                | `Servlet/3.1 JSP/2.3 (Apache Tomcat/8.5.78 Java/Oracle...)` | **信息泄露**：明确暴露了 Tomcat 版本 (8.5.78) 和 Java 版本 (1.8.0_151-b12)。攻击者可利用已知漏洞进行定向攻击。        |
| `Content-Type`                | `text/html; charset=UTF-8`                                  | 返回 HTML 页面而非 JSON 数据，属于**服务端渲染**的传统架构。                                               |
| `Access-Control-Allow-Origin` | `*`                                                         | **严重配置错误**：允许任意域名跨域读取该接口内容。结合 Cookie 中的 `SameSite` 默认行为，攻击者可构造钓鱼页面跨域窃取该 HTML 中的个人信息。 |
| `X-Frame-Options`             | `SAMEORIGIN`                                                | 良好实践，防止页面被其他域名 `<iframe>` 嵌套。                                                        |
| `X-XSS-Protection`            | `1; mode=block`                                             | 开启浏览器内置反射型 XSS 过滤。                                                                   |
| `X-Content-Type-Options`      | `nosniff`                                                   | 防止浏览器 MIME 类型嗅探攻击。                                                                   |

| 风险点                    | 描述                                                                                     | 危害等级   |
| :--------------------- | :------------------------------------------------------------------------------------- | :----- |
| **1. HTTP 明文传输**       | 整个会话过程（包括登录主页 `Referer` 和本接口）均在 HTTP 下进行。                                              | **严重** |
| **2. CORS 配置为 `*`**    | `Access-Control-Allow-Origin: *` 允许任意网站通过 AJAX 读取该页面内容。                                | **严重** |
| **3. 服务器版本暴露**         | `X-Powered-By` 头泄露 Tomcat 8.5.78 和 Java 1.8.0_151 版本。                                  | **中危** |
| **4. 直接 HTML 输出未转义风险** | 虽然本次请求未测试特殊字符，但若姓名、部门字段包含 `<script>` 标签且未做 HTML 实体编码，将导致存储型 XSS。                       | **中危** |
| **5. 跨域请求伪造 (CSRF)**   | 接口仅依赖 Cookie 鉴权，未使用 Anti-CSRF Token。攻击者可在第三方网站构造一个自动提交的表单，诱导已登录用户点击，从而修改用户信息（如果有修改接口）。 | **中危** |

| 组件 | 识别依据 |
| :--- | :--- |
| **前端** | Bootstrap 3 + jQuery + Tab 切换，服务端渲染 HTML。 |
| **后端** | Java Servlet / JSP (Tomcat 8.5)。 |
| **身份中间件** | `iPlanetDirectoryPro` Cookie 暗示 Oracle Access Manager (OAM) 11g/12c 或 Oracle Identity Governance 套件。 |
| **反向代理/负载** | `Server: none` 可能由 Nginx 或 F5 隐藏了真实 Web 服务器头。 |
