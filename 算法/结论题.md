> ## 题目描述
> 
> 一天，Colin 脑海中闪过一个绝妙的想法。
> 
> 给定一个长度为 $n$（下标从 $0$ 到 $n-1$）的、由小写字母组成的字符串 $S$。
> 
> 令 $S_i$ 表示 $S$ 中下标为 $i$ 的字符。例如，若 $S = \text{abc}$，则 $S_0 = \text{a}$，$S_1 = \text{b}$。
> 
> 令 $S[l: r]$ 表示 $S$ 中下标从 $l$ 到 $r$ 的子串。例如，若 $S = \text{abc}$，则 $S[1:2] = \text{bc}$。
> 
> 现在我们在字符串上进行一个游戏。你希望尽可能多地成功完成回合。
> 
> **每回合你需要执行以下操作：**
> 
> 1. 首先选择两个整数 $l, r$，满足 $0 \le l < r < n$，且 $S[l:r]$ 中至少包含两个不同的字符（即存在两个不同的整数 $x, y \in [l, r]$ 使得 $S_x \ne S_y$）。如果找不到这样的两个整数 $l, r$，则你在本回合失败，游戏结束。
> 2. 然后选择一个整数 $p \in [l, r]$，将 $S$ 中所有下标 $i \in [l, r]$ 对应的字符 $S_i$ 全部改为 $S_p$；随后本回合成功完成。
> 
> **示例：** 若 $S = \text{abc}$，选择 $l=1, r=2, p=2$，则本回合结束后 $S$ 变为 $\text{acc}$。
> 
> Colin 想问你：在字符串 $S$ 上最多能成功完成多少个回合？
> 
> Eva 觉得这对你来说太简单了，于是她把这道题升级为一道计数问题。令 $ans_S$ 表示字符串 $S$ 对应的上述问题的答案，你需要统计：**在所有长度为 $n$ 的、由小写字母组成的字符串中：**
> 
> 1. 有多少个字符串满足 $ans_S = 0$？
> 2. 有多少个字符串满足 $ans_S > 0$ 且 $ans_S$ 为有限值？
> 3. 有多少个字符串满足 $ans_S$ 为无穷大？
> 
> ---
> 
> ## 输入描述
> 
> 第一行一个整数 $n$（$2 \le n \le 10^9$），表示字符串的长度。
> 
> ## 输出描述
> 
> 一行三个整数：
> 
> - 第一个数表示满足 $ans_S = 0$ 的字符串数量，对 $10^9 + 7$ 取模。
> - 第二个数表示满足 $ans_S > 0$ 且 $ans_S$ 为有限值的字符串数量，对 $10^9 + 7$ 取模。
> - 第三个数表示满足 $ans_S$ 为无穷大的字符串数量，对 $10^9 + 7$ 取模。
> 
> ---
> 
> ## 示例 1
> 
> **输入：**
> ```
> 2
> ```
> 
> **输出：**
> ```
> 26 650 0
> ```
> 
> **说明：**
> 共有 26 个长度为 2 的字符串满足 $S_0 = S_1$，它们的 $ans = 0$。
> 
> 对于其余长度为 2 的字符串，容易看出你只能成功完成 1 个回合，因此它们的 $ans = 1$。

数据量太大了，无从直接下手。先看看具体的例子

对于 $n=2$ 的情况，
- 要不是两个字母相同，无法进行回合，$ans=0$
- 要不是两个字母不相同，进行了一回合之后又变成了第一种情况，$ans=1$

所以我们可以计算出 $n=2$ 时的字符串数量
- $26$
- $26^2-26=650$

对于 $n=3$ 的情况
- 考虑字母都相同的情况，比如 `aaa` ，这种情况有26个，都是 $ans=0$ 
- 考虑字母有两个不同的情况，比如 `abb`
	- 第一回合结束后，可以变成 `aab`
	- 第二回合结束后，可以变成 `abb`
	- 。。。回到了相同的局面
	- 这意味着 $ans=\infty$
- 考虑字母全都不同的情况，比如 `abc`
	- 第一回合结束后，可以变成 `abb`
	- 第二回合结束后，可以变成 `aab`
	- 。。。这和上面一样
	- 这意味着 $ans=\infty$

嗯。。。我们不难想到，所有 $n>2$ 的情况，都可以化为与 $n=3$ 类似的局面。
这意味着，对于所有 $n>2$ 的情况，他们的答案是：
- $26$
- $0$
- $26^n-26$

```cpp
#include <iostream>

using namespace std;

const long long MOD = 1000000007;

// 快速幂计算 (base^exp) % MOD
long long fast_pow(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n;
    if (!(cin >> n)) return 0;

    if (n == 2) {
        cout << "26 650 0\n";
    } else {
        long long total = fast_pow(26, n);
        // 注意：减法取模要先加 MOD 规避负数
        long long ans_infinite = (total - 26 + MOD) % MOD;
        
        cout << 26 << " " << 0 << " " << ans_infinite << "\n";
    }

    return 0;
}
```

> [!question] Palindromex
> **时间限制：** 每个测试用例 2 秒
> **内存限制：** 每个测试用例 256 兆字节
> https://codeforces.com/gym/692235/problem/D
> 
> Yousef 给了你一个包含 $2n$ 个整数的数组 $a$。每个整数 $x \in [0, n-1]$ 在数组中**恰好出现两次**。
> 
> 你的任务是找到一个**回文子数组** $a_l, a_{l+1}, \dots, a_r$，使得其 $\text{mex}(a_l, a_{l+1}, \dots, a_r)^{\dagger}$ 尽可能大。输出这个最大的可能值。
> 
> > *回文是指正读和反读都相同的字符串，例如 "z"、"aaa"、"aba"、"abccba" 都是回文，而 "codeforces"、"reality"、"ab" 不是回文。*
> >
> > † 一个整数数组的 **mex**（最小排斥值）定义为不在该数组中出现的最小非负整数。例如：
> > - $[2, 2, 1]$ 的 $\text{mex}$ 为 0，因为 0 不属于该数组。
> > - $[3, 1, 0, 1]$ 的 $\text{mex}$ 为 2，因为 0 和 1 属于该数组，但 2 不属于。
> > - $[0, 3, 1, 2]$ 的 $\text{mex}$ 为 4，因为 0、1、2 和 3 都属于该数组，但 4 不属于。
> 
> ---
> 
> ## 输入格式
> 
> - 第一行包含一个整数 $t$（$1 \le t \le 10^4$）—— 测试用例的数量。
> - 每个测试用例的第一行包含一个整数 $n$（$1 \le n \le 10^5$）。
> - 每个测试用例的第二行包含 $2n$ 个整数 $a_1, a_2, \dots, a_{2n}$（$0 \le a_i \le n-1$）。保证区间 $[0, n-1]$ 内的每个整数恰好出现两次。
> - 保证所有测试用例的 $2n$ 之和不超过 $2 \times 10^5$。
> 
> ## 输出格式
> 
> 对于每个测试用例，输出一个整数——任意回文子数组的最大 $\text{mex}$ 值。
> 
> ---
> 
> ## 样例
> 
> | 输入              |
> | --------------- |
> | 6               |
> | 4               |
> | 1 2 0 3 3 0 2 1 |
> | 2               |
> | 0 1 0 1         |
> | 2               |
> | 1 1 0 0         |
> | 3               |
> | 2 0 2 1 1 0     |
> | 4               |
> | 0 1 3 0 3 1 2 2 |
> | 3               |
> | 0 1 2 1 0 2     |
> 
> | 输出  |
> | --- |
> | 4   |
> | 2   |
> | 1   |
> | 1   |
> | 2   |
> | 3   |
> 
> ---

> ## 样例说明
> 
> - **第一个测试用例：** 唯一的最优选择是子数组 $a[1, 8] = [1, 2, 0, 3, 3, 0, 2, 1]$，它是回文且 $\text{mex}$ 为 4。
> - **第二个测试用例：** 其中一个最优选择是子数组 $a[2, 4] = [1, 0, 1]$，它是回文且 $\text{mex}$ 为 2。
> - **第三个测试用例：** 可以选择子数组 $a[3, 3] = [0]$，它是回文且 $\text{mex}$ 为 1。不存在 $\text{mex}$ 大于 1 的其他回文子数组。

最小的答案是 $1$ ，因为总是存在一个 $0$ ，而且我们可以将 $0$ 单独作为一个宫位子数组与 $\operatorname{mex} = 1$ 。
任何包含 $\operatorname{mex} \gt 1$ 的有效子数组都至少包含一个 $0$ 。因此，我们知道最佳子数组总是包含一个 $0$ 。
这意味着，我们可以根据0来分类。假设 x 是数组中第一个 0 的位置，而 y 是第二个 0 的位置：
- x为回文中心
- y为回文中心
- x和y的中心为回文中心
我们只需比较这三种情况的 $\text{mex}$ 即可

```cpp
#include <bits/stdc++.h>
using namespace std;

int n;
vector<int> v;

int solve(int l, int r) {
    set<int> s;
    for(int i = 0; i <= n; i++) s.insert(i);

    while(l >= 0 && r < 2 * n && v[l] == v[r]) {
        s.erase(v[l]);
        l--, r++;
    }

    return *s.begin();
}

int main() {
    int t;
    cin >> t;
    while(t--) {
        cin >> n;
        v = vector<int> (2 * n);
        for(auto &it : v) cin >> it;

        int x = -1, y;
        for(int i = 0; i < 2 * n; i++) {
            if(!v[i]) {
                if(~x) y = i;
                else x = i;
            }
        }

        cout << max({solve(x, x), solve(y, y), solve((x + y) / 2, (x + y + 1) / 2)}) << endl;
    }
}
```
