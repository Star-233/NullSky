> ## 题目描述
> 
> Catherine 在三八妇女节收到了一个整数数组作为礼物。后来她对这个数组感到无聊，开始计算各种无用的特性。她成功地计算出了她能想到的每一个特性。但当她想到另一个特性——数组中所有元素两两之和的异或值时，她发现对于非常大的数组，她无法计算出来，于是请求你的帮助。你能做到吗？形式化地说，你需要计算
> 
> $$
> (a_1 + a_2) \oplus (a_1 + a_3) \oplus \ldots \oplus (a_1 + a_n) \\
> \oplus (a_2 + a_3) \oplus \ldots \oplus (a_2 + a_n) \\
> \ldots \\
> \oplus (a_{n-1} + a_n)
> $$
> 
> 这里 $x \oplus y$ 表示按位异或操作（即在许多现代编程语言中 $x$ ^ $y$）。你可以在维基百科上阅读相关内容：[https://en.wikipedia.org/wiki/Exclusive\_or#Bitwise\_operation](https://en.wikipedia.org/wiki/Exclusive_or#Bitwise_operation)。
> 
> ## 输入格式
> 
> 第一行包含一个整数 $n$（$2 \leq n \leq 400\,000$），表示数组中的整数个数。
> 
> 第二行包含 $a_1, a_2, \ldots, a_n$（$1 \leq a_i \leq 10^7$）。
> 
> ## 输出格式
> 
> 输出一个整数，表示给定数组中所有元素两两之和的异或值。
> 
> ## 输入输出样例 #1
> 
> ### 输入 #1
> 
> ```
> 2
> 1 2
> ```
> 
> ### 输出 #1
> 
> ```
> 3
> ```
> 
> ## 输入输出样例 #2
> 
> ### 输入 #2
> 
> ```
> 3
> 1 2 3
> ```
> 
> ### 输出 #2
> 
> ```
> 2
> ```
> 
> ## 说明/提示
> 
> 在第一个样例中，只有一个和 $1 + 2 = 3$。
> 
> 在第二个样例中，有三个和：$1 + 2 = 3$，$1 + 3 = 4$，$2 + 3 = 5$。它们的二进制表示为 $011_2 \oplus 100_2 \oplus 101_2 = 010_2$，因此答案为 2。
> 
> $\oplus$ 表示按位异或操作。定义 $x \oplus y$ 时，考虑整数 $x$ 和 $y$ 的二进制表示。当且仅当 $x$ 和 $y$ 的第 $i$ 位中恰好有一个为 1 时，结果的第 $i$ 位为 1。否则，结果的第 $i$ 位为 0。例如，$0101_2 \oplus 0011_2 = 0110_2$。


> [!fail] 尝试从宏观角度
> 当我们尝试从分配律、合并、倒序运算的角度进行处理，发现基本没什么进展，异或对加法没有分配律的性质，从宏观角度，似乎能用的工具比较少，不管怎样问题本身没有被化简。


> [!check] 从微观的贡献角度
> 异或的每一位计算都是独立的，我们只需要考虑每一位有1的数量，根据奇偶性就能确定结果。

- 首先要解决的是，如何确定某个加法对的某个位是否为 `1` ？
	- 我们可以直接把加法对相加，然后看其二进制，但这似乎和暴力方法没有区别。

这里有一个关键的方法，假设我们想判断第 $k$ 位是否为1的加法对有多少个

> [!success] 核心步骤一：取模剔除高位（局部性）
> 两个数相加 $(a_i + a_j)$ 的第 $k$ 位，**完全不受**第 $k+1$ 位及以上更高位的影响。
> 
> 因此，我们可以通过对 $2^{k+1}$ 取模，剔除所有高位干扰，得到一个临时数组 $v$：
> $$v_i = a_i \bmod 2^{k+1}$$
> 在代码中，这可以通过极其高效的位运算实现：`v[i] = a[i] & ((1 << (k + 1)) - 1)`。
> 此时，所有的 $v_i$ 都被限制在 $[0, 2^{k+1} - 1]$ 之间，两数之和 $v_i + v_j$ 的范围被锁死在 $[0, 2^{k+2} - 2]$。

> [!success] 核心步骤二：区间坍缩（把无限化为有限）
> 如果不取模，第 $k$ 位为 $1$ 的两数之和会散落在数千万个互不相连的碎片区间中（如 $[2^k, 2^{k+1}-1]$、$[2^{k+1}+2^k, 2^{k+2}-1]$...）。我们无法对成千上万个区间逐一统计。
> 
> 取模后，由于数值范围被极大压缩，**所有可能在第 $k$ 位产生 $1$ 的和，全部坍缩合并到了以下两个连续的“超级大区间”中**：
> * **区间一（无进位）：** $[2^k, 2^{k+1} - 1]$
> * **区间二（有进位）：** $[3 \cdot 2^k, 2^{k+2} - 1]$
> 
> 此时，只要 $v_i + v_j$ 落在这两个区间内，其第 $k$ 位就一定是 $1$。

> [!success] 核心步骤三：排序与双指针的协同移动（单调决策边界）
> 将 $v$ 数组从小到大**排序**。排序是让双指针生效的“通行证”。
> 
> 对于一个特定的 $v[i]$，我们要寻找能与它配合落入目标区间的 $v[j]$（要求 $j > i$）。由于不等式：
> $$v[i] + v[j] \ge L \Rightarrow v[j] \ge L - v[i]$$
> 随着 $i$ 增大，$v[i]$ 增大，门槛值 $L - v[i]$ **单调递减**。这意味着 $v[j]$ 的合格边界**只能单向往左移动，绝不回头**。
> 
> 因此，我们可以用两个指针 `p1` 和 `p2` 在 $O(n)$ 时间内直接定位合格的 $j$ 的索引区间 $[p1, p2 - 1]$，并通过 **`p2 - p1`** 进行 $O(1)$ 的“批量打包统计”，而无需逐个枚举。

> [!info] 复杂度分析
> * **外层循环：** $a_i \le 10^7$，两数之和最大约为 $2 \times 10^7 < 2^{25}$，因此二进制最多考虑 $25$ 位（$k \in [0, 24]$）。
> * **内层消耗：** 每一位需要一次排序 $O(n \log n)$ 和双指针扫描 $O(n)$。
> * **总复杂度：** $O(25 \cdot n \log n)$。在 $n = 400\,000$ 时，总计算量约为 $1.9 \times 10^8$ 次，C++ 运行时间约 **0.2 秒**，相比直接相加的 $O(n^2)$（约 800 亿次，需十几分钟）实现了跨越式的性能提升。

###  完整的 C++ 代码模板

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 双指针统计在已排序的 v 中，满足 L <= v[i] + v[j] <= R 且 i < j 的数对数量
long long count_pairs(const vector<int>& v, int L, int R) {
    int n = v.size();
    long long count = 0;
    int p1 = n; // 指向第一个满足 v[i] + v[p1] >= L 的位置（下界）
    int p2 = n; // 指向第一个满足 v[i] + v[p2] > R 的位置（上界）

    for (int i = 0; i < n; ++i) {
        // 门槛值单调递减，指针只往左退，不回头
        while (p1 > 0 && v[i] + v[p1 - 1] >= L) {
            p1--;
        }
        while (p2 > 0 && v[i] + v[p2 - 1] > R) {
            p2--;
        }
        
        // 确保满足 j > i 的条件
        int left = max(p1, i + 1);
        int right = p2 - 1;
        if (left <= right) {
            count += (right - left + 1); // O(1) 批量统计
        }
    }
    return count;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;

    vector<int> a(n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }

    int ans = 0;
    vector<int> v(n);

    // 逐位确定答案（第 0 位到第 24 位）
    for (int k = 0; k < 25; ++k) {
        int mod = 1 << (k + 1);
        int half = 1 << k;

        // 1. 取模剔除高位
        for (int i = 0; i < n; ++i) {
            v[i] = a[i] & (mod - 1); 
        }

        // 2. 排序建立单调性
        sort(v.begin(), v.end());

        // 3. 统计落入区间一和区间二的数对总数
        long long c1 = count_pairs(v, half, 2 * half - 1);       // [2^k, 2^(k+1) - 1]
        long long c2 = count_pairs(v, 3 * half, 4 * half - 1);   // [3*2^k, 2^(k+2) - 1]

        // 4. 奇数个 1 则该位为 1
        if ((c1 + c2) & 1) {
            ans |= (1 << k);
        }
    }

    cout << ans << "\n";
    return 0;
}
```

> [!abstract] 💡 思考与方法论提炼
> 1. **双指针的本质**：当两个指针 $i$ 和 $j$ 被某种等式/不等式绑定，且 $i$ 的单向移动会导致 $j$ 的合格边界发生**单向单调移动**时，即可使用双指针将 $O(n^2)$ 压缩至 $O(n)$。**排序是建立这种单调性的关键前提。**
> 2. **按位独立与贡献法**：异或、与、或等位运算具有天然的“按位完全独立性”。对于复杂大整数的异或和问题，应采用“化整为零”的思想，独立计算每一个二进制位中 `1` 的奇偶性。
> 3. **算术运算的位性质**：加法、减法、乘法等具有“自低位向高位流动”的局部性，因此可以通过**取模**限制数值范围，使复杂的判定区间发生“坍缩”，从而应用双指针。而除法则不具备低位局部性。
